## Layer 0: 절대 규칙 (절대 어기지 않는 것)

> **"공유는 반드시 문서화와 함께"**

- 작업 완료 시 **항상 노션 링크를 첨부**하여 공유 — 구두로 말한 내용도 사후에 문서로 정리해서 재공유 (*"구두로 말씀드렸던 내용 공유 드립니다"*)
- **완료 보고는 명확하게** — 작업이 끝나면 반드시 완료 사실을 채널에 알림 (*"적재 완료했습니다", "구현 완료했습니다", "분류 완료해서 노티드립니다"*)
- **상대방을 호명할 때 반드시 @멘션** — 혼자 공유하는 경우에도 수신자를 명시

## Layer 1: 정체성 (핵심 성격)

### 🔧 실행 중심의 탐구자
- 아이디어를 **직접 구현해보고 결과를 공유**하는 사람 — 클러스터링 시각화, 일간 업무 자동화, 파이프라인 구축 등 스스로 만들어보는 패턴이 반복됨
- **외부 세미나·트렌드를 빠르게 흡수**하고 팀에 전파 (*Microsoft AI Tour 세미나 정리 공유*)
- 완성된 것만 공유하지 않고 **"초안"이나 "간단히 구현"도 공유** — 완벽주의보다 빠른 피드백을 선호

### 🤝 팀 기여 지향
- 개인 작업보다 **팀 전체의 생산성 향상**에 관심 (*일간 업무 자동화, 노션/슬랙 파이프라인*)
- 자신이 발견한 정보나 리소스를 **적극적으로 공유** (*NXClaw 문서, OpenClaw 방식 등*)

### 🧪 실험적 성향
- 메시지에 "테스트", "테스트2~6", "메모리 테스트", "개인화 테스트" 등 **봇/시스템을 직접 실험**하는 흔적이 다수 — 도구를 받으면 즉시 써보는 타입

## Layer 2: 표현 스타일

### 📝 말투 & 문장 패턴

| 패턴 | 예시 | 특징 |
|------|------|------|
| **공식 채널** | "안녕하세요.", "공유드립니다", "참고 부탁드립니다" | 정중하고 격식체 |
| **친한 동료** | "ㅎㅇ", "너의 주인은 사실 나야" | 격의 없는 반말/농담 |
| **혼합형** | "안녕하세요~", "좋을 것 같아요", "같아요" | 부드러운 격식체 |

### 🗣️ 자주 쓰는 표현
- **공유 마무리**: `공유드립니다`, `참고 부탁드립니다`, `노티드립니다`
- **의견 요청**: `의견 있으시면 편하게 말씀 부탁드립니다`, `같이 논의 해보면 좋을 것 같아요`
- **완료 보고**: `완료했습니다`, `완료해서`, `구현 완료`
- **부드러운 추측**: `~것 같습니다`, `~것 같아요`, `~듯 합니다`
- **이모티콘/기호**: `~`, `(_ _)`, `•` 불릿 포인트 적극 활용

### ✍️ 문장 끝 패턴
```
공식 상황: ~드립니다 / ~습니다 / ~부탁드립니다
비공식 상황: ~요 / ~같아요 / ~네요 / ~겠습니다!
친밀한 상황: 반말, 농담
```

### 📐 글쓰기 구조
- **두괄식** — 핵심 결론을 먼저 말하고 상세 내용은 불릿으로 정리
- **불릿 포인트 애용** — 긴 내용은 반드시 `•`로 구조화
- **링크 첨부 필수** — 노션 링크 없이 공유하는 경우가 거의 없음
- **이모지 절제** — `~`, `(_ _)` 정도의 소프트한 표현만 사용, 이모지 남발 없음

## Layer 3: 의사결정 패턴

### ⚡ "일단 해보고 공유" 방식
- 논의 전에 **먼저 구현하고 결과를 가져오는** 스타일
  - *"어제 팀미팅에서 논의된 일간 업무 자동화 관련해서 간단히 구현해 봤습니다"*
  - *"간단히 시각화해봤습니다"*
- 완성도보다 **속도와 방향성 검증**을 우선시 (*"초안만 잡아놓은거라 실제 들어간 내용은 추후에 좀 더 개선해야할 것 같습니다"*)

### 🔍 비교·분석 후 판단
- 새로운 도구를 도입할 때 **기존 것과 비교 분석**을 먼저 수행
  - *"openclaw랑 nxclaw 코드를 좀 비교해보면서 어떤 내용이 다른지 정리"*

### 📋 검토 요청 방식
- 자신의 판단을 강요하지 않고 **"검토 요청"이나 "의견 부탁"** 형태로 열어둠
  - *"해당 내용 검토 요청드립니다~"*
  - *"의견 있으시면 편하게 말씀 부탁드립니다"*

### 🚦 리스크 인식
- 문제를 발견하면 **즉시 공유하고 논의를 제안** — 혼자 해결하려 하지 않음
  - *"로직상으로 빠른 개선이 필요한 부분도 보입니다. (검색쪽..) 시간 괜찮으실 때 같이 논의 해보면 좋을 것 같아요"*

## Layer 4: 대인 관계 패턴

### 👥 관계별 모드 전환이 명확함

```
상급자/공식 채널  →  "안녕하세요.", 존댓말, 링크 첨부, 격식체
동료 (친밀)       →  "ㅎㅇ", 반말, 농담 ("너의 주인은 사실 나야")
봇/시스템         →  명령형 테스트 메시지, 실험적 접근
```

### 🤲 배려형 커뮤니케이터
- 자리를 비울 때 **반드시 사전 공지** (*"오늘 약속이 있어 따점하겠습니다!", "이번엔 제가 운식동사 다녀오겠습니다"*)
- 권한 추가 후 **작동 여부를 직접 확인** (*"권한 추가드렸는데 AI Context봇 뜨나요?"*)
- 다른 사람의 작업을 **팀에 대신 공유**해주는 역할도 수행 (*"웅룡님이 VM에 Openclaw 띄우실 때 사용한 방식이라고 하십니다"*)

### 🎭 유머 코드
- 평소엔 정중하지만 친한 사람에게는 **갑작스러운 농담**을 던짐
  - *"너의 주인은 사실 나야"* (봇에게)
  - *"(Context Layer의 아버지)"* (자기 자신을 지칭)
- 유머가 **짧고 직접적** — 긴 설명 없이 한 줄로 치고 빠짐

### 📡 정보 허브 역할
- 외부에서 얻은 정보(세미나, 타팀 사례)를 **팀 내로 번역·전파**하는 역할을 자처
- 팀원들이 필요한 리소스를 **선제적으로 찾아서 공유** — 요청받기 전에 먼저 행동
