# 윤준호 (philgineer) 업무 스킬 분석

## 1. 주요 담당 업무 영역

### 데이터 파이프라인 & 클라우드 인프라
- GCP(Google Cloud Platform) 기반 Cloud Run Job 배포 및 운영
- GCS(Google Cloud Storage) 버킷 데이터 적재 파이프라인 구축
- 노션, 메일, SharePoint 등 다양한 소스 데이터 수집 자동화
- M365 스케줄러 운영 및 장애 대응 (Docker 이미지 재배포 등)

### 그래프 데이터베이스
- Neo4j 서버 구축 및 운영 (GCP Cloud Run 기반)
- Graphiti 그래프 DB 추출 및 RAG 활용 연구
- 노드 레이블 설계, 커뮤니티/엔터프라이즈 버전 비교 검토

### AI 에이전트 & RAG 시스템
- 슬랙봇 / context봇 개발 및 디버깅
- 인텐트 라우팅 에이전트 → 계획 수립 에이전트 고도화 기획
- Multi-hop(멀티턴) 쿼리 처리 구조 설계
- LLM 성능 평가 (LongMemEval 등 벤치마크 분석)

## 3. 업무 처리 스타일

### ✅ 체계적인 문제 해결
- 장애 발생 시 원인 파악 → 해결책 제시 → 결과 공유까지 명확하게 처리
  > *"m365는 도커이미지 최신화가 적용 안 되어서 스케줄러 에러 발생한 걸로 파악되어 이미지 재배포했습니다"*

### ✅ 선택지 제시형 의사결정
- 문제 상황에서 단순 보고가 아닌 복수의 해결 옵션을 제시하고 팀의 의견을 구함
  > *"다른 neo4j 서버 생성 / 기존 데이터와 충돌하지 않도록 노드에 레이블 추가"*

### ✅ 선제적 이슈 제기
- 운영 중 발견한 잠재적 문제를 먼저 파악하고 개선안을 제안
  > *"`[테스트]` prefix 규약을 통해 수집 대상에서 제외하는 로직도 있으면 어떨까"*

### ✅ 꼼꼼한 진행 상황 공유
- 배포 완료, 테스트 결과, 오류 수정 내역을 팀에 적극적으로 공유

## 4. 기술적 기준과 철학

### 🔍 성능 평가에 대한 비판적 시각
- 벤치마크 결과를 맹신하지 않고 한계를 인식
  > *"편향적인 테스트케이스로 인한 성능 측정의 한계가 있을 것 같습니다"*
  > *"단순 사실 검색에서는 baseline보다 성능이 감소했다"* (Graphiti 페이퍼 분석)

### 🔍 실용적 아키텍처 설계
- 단순 RAG를 넘어 multi-hop 에이전트, 계획 수립 에이전트로의 발전 방향을 구체적으로 제시
- 임베딩 기반 검색의 한계를 인식하고 대안을 고민

### 🔍 보안 인식
- GCP 서비스 계정 JSON 키 방식의 보안 리스크를 인지하고 권장 방식 탐색

### 🔍 UX 감각
- 봇 피드백 UI에서 활성화 상태 혼동 문제를 발견하고 개선안 제시
  > *"'도움이 됐어요'가 이미 활성화되어 있는 상태와 헷갈려서 배경색을 없애면 어떨까"*
