# 이창영3 (leecy) 업무 스킬 분석

## 1. 주요 담당 업무 영역

### 🎯 AI 컨텍스트 팀 코디네이터/PM 역할
- **메이플 아시아실 × AI Context 팀** 협업 창구 역할
- 외부 팀과의 미팅 일정 조율, 회의실 예약, 안건 사전 준비
- 팀 주간보고 작성 및 취합 주도
- 데일리 스크럼 운영 및 팀 업무 분배 정리

### 📊 데이터 수집 및 분석 기획
- TLM(Translation/Log Management 추정) 데이터 추출 및 공유
- 노션/슬랙 채널 기반 실·팀 단위 데이터 수집 구조 설계
- RAG Engine 관련 문의 대응
- Graph DB 구축 업무 기획 참여

### 🤖 AI 도구 도입 및 운영
- **오픈클로(OpenClaw)** 프로그램 운영 (모집, 설문, 가이드 관리)
- **n8n** 워크플로우 팀 계정 운영 및 팀원 초대
- AI 봇(슬랙 봇) 테스트 및 배포 검토
- Claude Code, AI Hub 등 사내 AI 도구 도입 지원

## 3. 업무 처리 스타일

### ✅ 선제적·능동적 처리
- 권한 문제를 스스로 파악하고 다음 액션을 즉시 공지
  > *"내일 요청드려보겠습니다"*
- 개인 용도로 만든 도구도 팀에 공유하는 오픈 마인드
  > *"개인적으로 쓰려고 만들었던건데 참고 되실까 하여 공유드립니다"*

### 📋 체계적인 정보 정리
- 회의 전 사전 자료 공유 및 질문 리스트 취합
- 팀원별 업무를 구조화하여 정리 후 공유
- 환경별(dev/stage/live) URL 명확히 구분하여 공유

### ⚡ 실용주의적 문제 해결
- 도구가 기대에 못 미치면 직접 처리
  > *"오픈클로가 너무 멍청해서 그냥 제가했습니다"*
- 완벽하지 않아도 일단 테스트하고 피드백 수렴

## 4. 기술적 기준과 철학

### 🔒 보안 의식
- SSL 인증서 관련 가이드 직접 제공 검토
- 비정상 로그인 탐지 메일 즉시 확인 요청
- 보안 우회 설정 안내에 대한 문제의식 표현
  > *"ssl 인증 무시 설정으로 안내해주셨는데 (그러셔도 되나)"*
- Windows 환경에서 보안까지 고려한 도구 사용법 질의

### 🏗️ 확장성 고려한 설계 지향
- 팀 단위 → 실 단위로 데이터 수집 영역 확장 기획
- 채널-팀 맵핑 테이블 설계로 구조적 확장성 확보
- 봇 채널을 고정화하여 반복 수정 부담 최소화 제안

### 🧪 단계적 검증 문화
- 실 채널 오픈 전 테스트 단계 명확히 구분
  > *"아직 열겠다고 한건 아니라서 우려되시는 점 있으시면 미뤄도됩니다"*

## 5. 커뮤니케이션 패턴

### 💬 스타일
- **친근하고 부드러운 톤** 유지 (`~님`, `~부탁드립니다`, 이모지 활용)
- 늦은 시간 메시지 시 **양해 먼저 구함**
  > *"늦은시간에 죄송합니다"*
- 강요 없이 **자율성 존중**하는 표현 선호
  > *"자유롭게 말씀 부탁드립니다"*

### 📢 정보 전달 방식
- 백틱(`` ` ``)으로 제목/키워드 강조
- 노션 링크를 적극 활용한 문서 기반 소통
- `<!here>` 태그로 팀 전체 공지 시 명확한 사유 제시

### 🤝 협업 태도
- 타 팀/회사 정보도 팀에 적극 공유 (업계 동향 공유)
- AI 봇을 직접 테스트하며 버그 발견 시 즉시 리포트
- 팀원 업무 가시화를 위한 회의록 작성 주도
